Selamat datang di halaman Expert

Oke, sampai sini berarti kalian udah lewat tahap “bikin jaringan bisa jalan” dan udah paham cara mikrotik berpikir dari sisi dasar sampai menengah. Nah di level expert ini, tugasnya bukan cuma ngatur supaya internet nyala, tapi supaya internetnya juga “punya aturan main”. Jadi sekarang kita bakal belajar cara bikin mikrotik bertindak kayak satpam yang tegas: ada yang boleh, ada yang ditahan, dan ada yang dibuat seperti nggak pernah ada.

Dua senjata yang bakal kita pakai di sini adalah DNS Static dan Web Proxy. DNS Static itu kerjanya di tahap “penunjuk jalan” (dia bantu memutuskan alamat tujuan dari nama domain), sedangkan Web Proxy itu pegangannya lebih ke “lalu lintas web”-nya (jadi mikrotik bisa ngatur permintaan saat client lagi browsing atau download). Kombinasi dua fitur ini bikin kontrol kamu makin rapi: blokir website bisa terasa lebih cepat dan blokir file tertentu bisa lebih spesifik. Gaskeun ke dropdown di bawah, biar kamu punya bahan yang siap dipraktekin.

Di sub-modul ini kamu bakal belajar cara “mengelabui” akses website dengan memanfaatkan DNS Static. Intinya, ketika client minta akses ke sebuah domain, mikrotik ikut campur di bagian penerjemahan nama domain menjadi alamat IP. Nah, kalau domainnya kita arahin ke alamat yang tidak semestinya (misalnya IP blackhole atau alamat yang tidak bisa dijangkau), maka website tujuan akan terasa seperti benar-benar mati. Serius, rasanya kayak kamu ngunci pintu sebelum orang sempat nginjek gagang pintunya.

Supaya aturan DNS Static ini kebaca oleh client, pastikan DNS yang digunakan client memang mengarah ke Mikrotik. Biasanya ini beres kalau kamu sudah mengatur DHCP agar mengirim DNS server mikrotik (opsi DNS di DHCP/option DNS), atau kamu set DNS manual di perangkat client ke alamat IP mikrotik. Setelah itu, tinggal masuk menu IP > DNS > Static lalu tambahkan entri untuk domain yang mau diblokir beserta alamat pengalihan/blackhole-nya. Setelah selesai, coba tes dari client: akses domain yang diblokir harusnya gagal, sementara domain lain tetap jalan normal.

Kalau DNS Static lebih fokus ke “nama domain-nya” duluan, Web Proxy itu fokusnya ke kontrol trafik web yang lewat dari client. Jadi kamu bisa bikin mikrotik bertindak sebagai perantara: saat user melakukan permintaan browsing atau download file, aturan proxy bisa dimanfaatkan untuk menolak file-file tertentu. Dengan kata lain, webnya mungkin masih bisa dibuka, tapi request download yang mengandung file berbahaya (misalnya ekstensi yang tidak diinginkan) bisa kamu tahan. Santai, tapi tetap tegas.

Konsep setingnya biasanya begini: aktifkan Web Proxy di Mikrotik, tentukan port/proxy settings-nya, lalu pastikan client memakai proxy itu (biasanya lewat pengaturan manual di browser atau lewat konfigurasi DHCP supaya client otomatis diarahkan ke proxy). Setelah proxy aktif, masuk ke bagian aturan akses (Web Proxy Access) dan buat rule yang isinya menolak request berdasarkan pola URL/ekstensi file atau tipe konten yang terdeteksi oleh proxy. Contoh praktis yang sering dipakai: blok download dengan ekstensi seperti .exe, .zip, atau .rar. Setelah rule diterapkan, tes dengan mencoba download dari client—yang diblokir harus gagal, yang aman harus tetap bisa.
